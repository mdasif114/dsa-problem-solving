problem starts from its starter file. Record time, score, mistakes, and next
review in `progress.md`.

## Spaced-repetition rule

For every Must problem:

1. First attempt on its scheduled day.
2. Explain the invariant or data structure the next day.
3. Re-solve without notes after 3–5 days.
4. Re-solve under time pressure after 8–12 days if the first score was below 3.

## Adaptation rules

- If score is below 3 for correctness or testing, replace the next Additional
  problem with a re-solve.
- If coding exceeds 35 minutes but the approach was correct, practise a simpler
  variant and Java collection syntax rather than a harder algorithm.
- If clarification or communication scores below 3, spend the next mock's first
  five minutes writing and speaking the contract before touching code.
- If three Must problems remain below 3 on 7 August, drop P31–P40 entirely.
- Do not compensate for a missed day by sacrificing sleep or doing two hard
  mocks consecutively.
